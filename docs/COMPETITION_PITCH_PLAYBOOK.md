# TalentScope 竞赛答辩作战手册

## 1. 8 页答辩结构（可直接做 PPT）

### 第 1 页：一句话定位（开场 20 秒）
- 标题：TalentScope 不是简历打分工具，而是招聘协同决策系统
- 你要讲：
  - 我们解决的不是“看简历慢”这个点问题，而是“关键岗位补位慢、空岗风险高、反馈不闭环”的团队问题。
  - 系统目标：提效、提质、降风险。
- 建议画面：
  - 三个关键词大字：提效 / 提质 / 降风险
  - 一张系统主界面截图

### 第 2 页：业务痛点（30 秒）
- 标题：传统流程的三大断点
- 你要讲：
  - 断点 1：筛选依赖个人经验，标准不统一。
  - 断点 2：候选人反馈分散，无法形成复盘数据。
  - 断点 3：关键岗位后备池不可视，空岗预警滞后。
- 建议画面：
  - 旧流程泳道图（HR、面试官、用人经理各自分散）

### 第 3 页：方案全景（35 秒）
- 标题：从评分到治理的闭环架构
- 你要讲：
  - 稳定规则评分 + LLM 解释。
  - 模板库治理 + 回归检查。
  - 反馈闭环 + 趋势看板 + 补位任务清单。
- 建议画面：
  - 输入（JD/简历）-> 匹配评分 -> 人工复核 -> 反馈沉淀 -> 趋势与预警 的闭环图

### 第 4 页：TEAM 能力（40 秒）
- 标题：从 TEAM 维度看系统成熟度
- 你要讲：
  - Technology：可解释、可复现、多模型可切换。
  - Execution：有脚本、有报表、有持续迭代。
  - Adoption：HR 场景可用，直接服务补位决策。
  - Management：权限分层、模板规范、风险预警。
- 建议画面：
  - 四象限图，分别对应四个字母

### 第 5 页：关键功能演示（60 秒）
- 标题：5 步完成一个岗位补位闭环
- 你要讲：
  - 选择岗位模板并应用硬性条件。
  - 执行候选匹配，查看证据与追问建议。
  - 形成 A/B/C 后备池，识别岗位连续性风险。
  - 在人才流失预警面板生成补位任务清单并下载。
  - 回写反馈，进入趋势看板复盘。
- 建议画面：
  - 主页面连续截图，按 1 到 5 编号

### 第 6 页：治理与安全（25 秒）
- 标题：可治理，才可落地
- 你要讲：
  - 管理员与 HR 角色分层：HR 只读筛选，管理员后台控制。
  - 首次登录强制改密、账号启停与重置。
  - 模板规范文档化、回归脚本化，降低漂移风险。
- 建议画面：
  - 权限矩阵表（角色 × 操作）

### 第 7 页：业务价值量化（35 秒）
- 标题：从“能用”到“有结果”
- 你要讲：
  - 建议固定 6 个评审指标：
    - 初筛耗时
    - 匹配命中率
    - 面试后不符率
    - 反馈覆盖率
    - 高风险岗位占比
    - 平均补位周期
  - 给出试点前后对比（即使是小样本，也要有趋势）。
- 建议画面：
  - 前后对比柱状图 + 趋势折线图

### 第 8 页：落地计划与扩展（25 秒）
- 标题：两周即可从试点到部门推广
- 你要讲：
  - 第 1 周：岗位模板校准 + 历史样本回测。
  - 第 2 周：上线试点 + 周报复盘。
  - 后续：与内部招聘系统接口联动，实现自动任务流。
- 建议画面：
  - 两周甘特图

## 2. 3 分钟速讲稿（可直接念）

我们团队做 TalentScope，不是为了做一个会打分的模型，而是要解决招聘团队真实的协同问题：关键岗位补位慢、空岗风险高、反馈不闭环。

传统流程里，HR、面试官、用人经理各自有判断标准，导致效率和一致性都不稳定。我们把流程做成闭环：先用稳定规则生成可复现分数，再由模型补充解释、风险提示和追问建议，帮助人工复核更快更准。

系统不仅有匹配结果，还有管理能力。我们提供岗位模板治理、模板回归检查、人才流失预警、补位任务清单导出，以及反馈趋势看板。也就是说，系统输出的不只是分数，而是可执行动作：哪个岗位有缺口、缺口多少、谁负责、什么时候补齐。

在安全与治理上，我们做了角色分层。HR 用户只负责浏览和筛选，管理员可以维护后台配置；同时支持账号启停、密码重置和首次登录强制改密。

从 TEAM 角度看，Technology 上我们可解释、可复现；Execution 上我们有脚本评测和持续迭代；Adoption 上直接服务 HR 补位场景；Management 上有权限和治理机制。下一步我们会持续跟踪六个核心指标，确保系统不只是演示好看，而是持续创造业务价值。

## 3. 评委高频问题与建议回答

### 问题 1：你们和普通 ATS 筛选插件有什么区别？
- 回答重点：
  - 我们不是单次筛选，而是全流程治理。
  - 有模板回归、反馈闭环、风险预警、任务清单导出。
  - 能沉淀团队共识并持续优化。

### 问题 2：模型不稳定怎么办？
- 回答重点：
  - 分数由本地规则计算，保证稳定复现。
  - 模型主要负责解释和建议，不直接决定录用。
  - 人工复核是强制环节。

### 问题 3：如何衡量成效？
- 回答重点：
  - 用固定六项指标做前后对比。
  - 每周生成趋势报告，持续验证改进效果。

### 问题 4：如何保障安全和权限？
- 回答重点：
  - 角色分层，最小权限。
  - 账号管理与强制改密。
  - 配置与模板可追踪。
